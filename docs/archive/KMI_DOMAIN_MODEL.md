# KMI DOMAIN MODEL

## Core entities

## 1) Grant categories
Examples: handrit, þróun, framleiðsla, eftirvinnsla, kynning, and other support.
Each grant should capture purpose, eligibility summary, best stage, deadlines, and traceable source metadata.

## 2) Documents
Application artifacts such as script, budget, finance plan, producer statement, director statement.
Each document needs:
- purpose,
- what it must prove,
- common weaknesses.

## 3) Criteria
Evaluation axes used in decision-making (e.g., artistic strength, feasibility, positioning), including red flags and evidence examples.

## 4) Funded projects / allocations
Historical records linking project title, year, amount, category, and company/people fields.
Used for pattern exploration, not for deterministic prediction.

## 5) Companies
Production entities connected to projects and allocations.

## 6) People
Writers, directors, producers, and other creative/production roles.

## 7) Producer-owned projects
Internal project records, stage, team, rights status, target grant, and notes.

## 8) Applications
Submission instances for a project+grant pair with status, result, and feedback.

## 9) Sources
Every extracted claim should link to sources with URL, title, checked dates, type, confidence, and notes.

## 10) Project readiness
Derived view combining project docs + grant requirements + criteria to highlight readiness score and risk flags.
Heuristic only; not official KMÍ methodology.
