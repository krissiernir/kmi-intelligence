# PRODUCT BRIEF

## Producer problem
KMÍ applications are trust packages, not just forms. A producer must demonstrate:
1) artistic strength
2) production execution credibility

Application work is fragmented across notes, spreadsheets, PDFs, and memory, which increases strategic mistakes.

## KMÍ knowledge problem
Critical knowledge is distributed across multiple KMÍ pages and historical allocation records. Producers need a structured way to connect:
- grant category fit,
- required vs strategically useful documents,
- evaluation logic,
- historical funding signals,
- readiness gaps.

## Why simple notes are not enough
Simple notes cannot reliably answer:
- Which grant fits this project now?
- Which documents are mandatory vs strategically important?
- What evidence each document must prove?
- What application risks a skeptical reviewer would spot?

## Tool objective
Build a local-first intelligence workspace that helps producers make better grant decisions through:
- grant decision support,
- document checklist strategy,
- allocation pattern analysis,
- project readiness scoring (heuristic, non-official),
- reusable AI brief prompts (without API coupling yet).
